How to Play:

  • The goal of the game is to turn off all the lights.
  • Each square on the grid represents a light that can be either "on" (black) or "off" (white).
  • Click on a square to toggle its state (from white to black or vice versa).
  • When you click a square, it will toggle its own state as well as the state of adjacent squares (up, down, left, right).
  • Keep clicking squares to turn off all the lights and win the game!
  • The puzzle starts in a random, but solvable configuration.

Addendum

This Lights Out game was designed as a 5x5 grid, where each square can be toggled by clicking it. The puzzle starts randomly but is solvable, and the player must click the squares in the right order to turn off all the lights.

Challenges Faced:

  • Generating a Solvable Puzzle: One of the main challenges was to generate a random, but solvable puzzle configuration. Simply randomizing the state of the squares could create unsolvable configurations. I had to implement a logic where each random configuration could be reversed to a solution, ensuring solvability.
  • Handling Adjacent Squares: The game rules required that when a square is clicked, it toggles its own state and also the state of the adjacent squares (up, down, left, right). Ensuring that these adjacent squares are properly toggled, and correctly implementing the logic, was a challenge.
  • Win Condition and Pop-Up Alert: Implementing the win condition required checking whether all the squares were turned off. This was done by checking the state of all squares after each move. Once the puzzle was completed, a pop-up alert was displayed. Handling this logic required working with JavaScript intervals and event listeners effectively to check game completion status.
  • Design and Responsiveness: I also focused on the design of the game, ensuring that the game grid was centered on the page, and that the instructions were placed next to the grid. Making the page responsive, so that it looks good on various screen sizes, was another challenge that required adjustments in the CSS layout.
  • Timer and Move Counter: Adding the timer and the move counter involved setting up a JavaScript interval to track time as the player made moves. The timer needed to reset at the start of each new game, and the move counter updated whenever the player made a valid move by clicking a square.
  • Game Restart: Another challenge was implementing the restart functionality. I added a button that resets the grid to a new solvable random configuration, resets the timer, and the move counter to start fresh.
  • Testing and Debugging: Testing the game for bugs was an important step. I encountered issues with event listeners not firing correctly in some scenarios and had to debug the code to ensure that clicking on the squares was always toggling the correct squares and states.

Despite the challenges, this project helped me learn more about JavaScript event handling, DOM manipulation, and logic implementation. It was an interesting experience to integrate all the pieces and create a fun and interactive game.
